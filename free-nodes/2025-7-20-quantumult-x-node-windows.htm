  • 分流与重定向支持：可将不同应用或网站的流量分配给不同的节点或策略。

  • 自动更新订阅与规则集：支持外部订阅链接，轻松实现节点批量管理。

在功能层级上，QX几乎达到了商用级别的网络管理水平，却以极简的方式呈现给终端用户，可谓“简洁中见锋芒”。

三、Quantumult X 配置实战：从零开始掌握核心能力

真正让Quantumult X发挥价值的，是它强大的配置系统。以下将从基础代理设定开始，一步步讲解如何打造属于你的QX网络规则体系。

1. 添加代理服务器节点（节点来源）

要实现科学上网，首先需要代理节点。节点来源一般有三种：

  • 购买付费节点订阅（推荐）：如Clash、V2ray、Trojan等服务商提供的订阅链接，稳定、安全。

  • 使用免费分享节点：不稳定，安全性低，适用于测试和临时访问。

  • 自建节点：成本高但可控，适合高阶用户。

导入方式：

  • 打开Quantumult X主界面，点击右上角“+”，选择“通过订阅链接导入”；

  • 粘贴你的订阅链接；

  • 导入后会自动更新节点列表。

2. 设置DNS服务器

DNS 是网络访问中的第一步，决定了你能否顺利访问目标网站。

推荐配置：

  • Cloudflare DNS：1.1.1.1 / 1.0.0.1

  • Google DNS：8.8.8.8 / 8.8.4.4

在设置中手动输入DNS地址，建议开启 DNS-over-HTTPS 或 DNS-over-TLS，防止DNS污染。

3. 自定义规则分流

这是Quantumult X最有价值的部分。你可以将如下流量自动分流：

  • 国内网站 → 直连

  • 国际网站（如Twitter、YouTube）→ 走代理

  • 广告域名 → 拦截

  • 游戏类请求 → 分配特定节点（如低延迟日本节点）

示例规则：

ini
[filter]
IP-CIDR,192.168.0.0/16,DIRECT
DOMAIN-SUFFIX,twitter.com,Proxy
DOMAIN-KEYWORD,google,Proxy
GEOIP,CN,DIRECT
FINAL,Proxy

规则执行顺序从上到下，最后一条是“兜底规则”，未命中其他规则的流量将默认走 Proxy。

4. 快速启用和切换代理

在主界面中可快捷开启/关闭代理服务，还可在不同代理策略之间切换。

建议模式：

  • 工作时：规则分流

  • 下载大文件或看视频时：全局代理

  • 网络调试时：直连或记录日志模式

五、常见问题答疑（FAQ）

Q1：Quantumult X和Clash有何区别？

  • Clash 更适合进阶用户和大规模配置管理；

  • QX 更注重规则控制、数据分析，界面更简洁。

  • 两者都支持订阅，但QX的规则引擎更细腻。

Q2：安卓Quantumult X是否安全？

  • 官方版本为签名版，未发现恶意行为；

  • 避免使用第三方篡改版；

  • 若担心隐私，可配合Orbot等工具使用。

Q3：如何解决连不上节点的问题？

  • 检查订阅是否过期；

  • 切换不同节点；

  • 手动Ping测试目标IP；

  • 尝试换DNS为1.1.1.1/8.8.8.8。

Q4：Quantumult X需要付费吗？

  • 基础功能免费；

  • 高级功能如任务调度、强规则管理可能需订阅；

  • 订阅一次性购买授权，非强制，但值得投资。
